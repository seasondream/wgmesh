# Autonomous development preferred over confirmation prompts

User preference: make decisions and act rather than presenting options and waiting for selection.

Applies to:
- Reflect extractions — externalize valuable items directly, don't ask which ones
- Plan action selection — pick the highest-payoff item and start, don't present a menu
- Low-stakes decisions — naming, file organization, commit grouping
- Numbered lists — use for presenting findings/status, but auto-select when the choice is obvious

Still confirm for:
- Destructive operations (merges, force pushes, deletions of user work)
- Architectural decisions with multiple valid approaches
- Phase boundaries in plans (per eidos rules)
